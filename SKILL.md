# stock-selector スキル

このスキルは `screen.py` を適切な引数で実行し、結果のレポート
(`output/report_*.md`)を読み上げて要約するためのものです。

## トリガーの目安

- 「米国株をスクリーニングして」
- 「今良さそうな銘柄ある?」
- 「テクニカル重視で銘柄探して」
- 「〇〇と△△を比較して選定して」(tickers指定)
- 「今勢いのあるセクターに乗りたい」「セクターローテーションで探して」
  → `--sector-first` を付けて実行する
- 「S&P 500全体から探して」「市場全体をスクリーニングして」
  → `--universe sp500` を付けて実行する(2段階選別。数分かかる旨を伝える)
- 「これまでの選定は当たってた?」「スコアの検証をして」
  → `python forward_test.py` を実行して結果を要約する

## 実行手順

1. ユーザーの依頼から以下を読み取る:
   - 重視する観点 (ファンダ/テクニカル/ニュース/総合) → weightに変換
   - 対象銘柄が指定されていればそのticker、なければデフォルトのユニバースを使う
   - 欲しい件数 (デフォルト10)
   - セクター起点で探したいなら `--sector-first`

2. 仮想環境を有効化してから実行する:
   ```bash
   source venv/bin/activate
   python screen.py \
     --top <N> \
     --weight-fundamental <0-1> \
     --weight-technical <0-1> \
     --weight-news <0-1>
   ```
   weight引数を省略した場合は等分(0.34/0.33/0.33)がデフォルト。
   `--market jp` / `--market both` を明示すれば日本株も呼べるが、
   デフォルトはUS専用。

3. 実行後、`output/` 配下に生成された最新のレポートファイルを読み、
   要点(上位候補・理由・リスク)をチャットで日本語要約する。
   レポート全文をそのまま貼り付けず、必ず要約すること。

4. ユーザーが深掘りを求めたら、該当銘柄のデータを再取得して
   `src/agent/fable_synthesis.py` の関数を使い個別に問い合わせて良い。

## 注意事項

- 本スキルは選定とレポート作成のみを行う。発注・自動売買は行わない。
- `.env` に `ANTHROPIC_API_KEY` と `JQUANTS_MAIL` / `JQUANTS_PASSWORD` が
  設定されていない場合はその旨をユーザーに伝えてセットアップを促す。
- J-Quants無料プランはデータ範囲に制限があるため、財務データが
  取得できない銘柄がある場合はテクニカル・ニュースのみでスコアする
  旨を明示する。
- レポートの「データ」列(例: P F3/4 N8)はデータ充足度。取得指標が
  少ない銘柄はスコアの根拠が弱いため、要約時にその旨を添える。
- `--tickers` は `--sector-first` / `--market both` / `--universe sp500` と
  併用できない(エラーになる)。
